# ─────────────────────────────────────────────────────────────────────────────
# 一、美术风格总定义
# ─────────────────────────────────────────────────────────────────────────────
# 风格：「冷峻像素」—— 偏暗饱和、蓝白冰雪主色调、硬像素无抗锯齿
#
# 色板：
#   主色   #B8D8FF（冰蓝高光）、#4E7CA1（中调冰面）、#1A2E42（阴影）
#   暖点缀 #E8C87A（斜阳雪光）   —— 仅用于主角头部装备/眼镜镜片
#   危险色 #E84040（血液/受击）、#FF9020（火焰/炸弹）
#   中性   #2A2A33（轮廓线）、#FFFFFF（雪花高光）
#
# 像素密度：
#   基础格子 = 16×16px → 所有碰撞体均对齐 16px 网格
#   角色     = 32×48px（宽×高，2格宽3格高）
#   大型Boss = 64×64px 或 96×96px
#   地形瓦片 = 16×16px
#
# 渲染设置（已在 project.godot 写入）：
#   textures/canvas_textures/default_texture_filter = 0（Nearest，保持像素锐利）
#
# 轮廓线规则：
#   - 角色 & 敌人：2px 深色外描边（#2A2A33），内无描边
#   - 地形：1px 描边即可
#   - UI 图标：1px 白描边（在深色背景下）
#
# 光影约定：
#   光源方向 = 左上 45°（所有元素统一）
#   影子色 = 基础色加深 30% + 蓝偏移 +20
#   高光色 = 基础色提亮 40%，最多 1~2 像素宽

# ─────────────────────────────────────────────────────────────────────────────
# 二、玩家动作组 & 动画规格
# ─────────────────────────────────────────────────────────────────────────────
# 文件路径：assets/sprites/player/player_sheet.png
# 格式：单张横向 spritesheet，每帧 48×48px（含武器挥动时的额外宽度）
# 注：角色身体占 32×48，左右各留 8px 给武器/特效
#
# ┌──────────────────────────────────────────────────────────────────────┐
# │ 动画名      帧数  FPS  循环  起始列  描述                            │
# ├──────────────────────────────────────────────────────────────────────┤
# │ idle         4    8    ✓    0       站立呼吸（轻微上下浮动 2px）     │
# │ run          6    12   ✓    4       小步快跑（适用于慢速地面移动）   │
# │ ski          6    14   ✓    10      滑雪姿势（身体前倾，雪板吃雪）   │
# │ jump_rise    3    10   ✗    16      起跳+上升（跳跃蓄力→腾空）      │
# │ jump_fall    2    8    ✓    19      下落（循环直到落地）             │
# │ land         2    12   ✗    21      落地缓冲（播放一次后→idle/run）  │
# │ attack       5    18   ✗    23      横向斩击（第3帧判定框激活）      │
# │ ski_attack   4    16   ✗    28      滑行攻击（前倾斩，不停步）       │
# │ skill_cast   4    14   ✗    32      冰刃投掷出手（第2帧发射）        │
# │ hurt         3    14   ✗    36      受击后仰（播放完毕回idle）       │
# │ dead         5    10   ✗    39      倒地死亡（最后一帧定格）         │
# └──────────────────────────────────────────────────────────────────────┘
#
# 动画过渡规则（AnimationPlayer 中用 Queue/Blend 实现）：
#   idle       → run/ski        : 任意时刻，有水平输入即切换
#   run        → ski            : 速度 ≥ 300px/s 时自动切换
#   ski        → run            : 速度 < 300px/s
#   任意地面   → jump_rise      : 按跳跃键，立即中断
#   jump_rise  → jump_fall      : 帧播完 OR velocity.y > 0
#   jump_fall  → land           : 落地（is_on_floor）
#   land       → idle/run/ski   : land 播完即切换（0.1s）
#   任意       → attack         : 按攻击键立即中断（有冷却）
#   attack     → ski_attack     : 在滑行中攻击自动选 ski_attack
#   attack末帧 → idle/run/ski   : 自动恢复
#   任意       → skill_cast     : 按技能键且冰刃已解锁
#   任意       → hurt           : take_damage() 调用时中断当前动画
#   hurt末帧   → idle           : 恢复
#   任意       → dead           : die() 调用时中断，dead 不循环不恢复
#
# 代码接入点：
#   scripts/player/Player.gd 中 _play_anim(name) 函数
#   → anim.play(name)（AnimationPlayer 节点名 = "AnimationPlayer"）
#   AnimationPlayer 需在 Player.tscn 中创建并按上表设置每个 Animation track

# 四、地形瓦片规格
# ─────────────────────────────────────────────────────────────────────────────
# 所有 tileset：16×16px/tile，PNG，无抗锯齿
# 每个 tileset 至少包含以下 tile 类型（用不同颜色/纹理区分）：
#
# 类型        外观要求
# ground      主地面，上表面有轮廓线，侧面有暗部
# slope_l     左坡（↗），斜面有竖条纹路
# slope_r     右坡（↘），斜面有竖条纹路
# platform    跳跃平台（薄板，仅上表面有碰撞）
# wall        竖向实心墙，侧面纹理较密
# ice_patch   透明度 30% 的蓝色覆盖层（叠在 ground 上）
# bg_detail   装饰瓦块（无碰撞，挂崖壁石/冰柱/雪堆）
#
# 各区域色调：
#   TestMap           白雪蓝影，清晨光感，主色 #D8EAF5
#   GlacierMaze       蓝绿冰面，透明感，主色 #6EC4D8
